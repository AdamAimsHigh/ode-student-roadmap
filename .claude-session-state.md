## 2. Architecture State

### 2.1 Runtime data loading model (critical constraint)
The app can be opened from `file://`, where `fetch()` of local JSON is blocked. Therefore
ALL data is delivered as plain `<script>` globals, never fetched:
- `app/js/curriculum-data.js` defines `const CURRICULUM_DATA = {...}`, then derives
  `const CURRICULUM = CURRICULUM_DATA.curriculum` and `const ALL_MODULES = [...]`.
- `app/js/quiz-data.js` defines `const QUIZ_DATA = {...}`.
- `app/data/curriculum.json` and `app/data/quizzes.json` are stale documentation mirrors,
  NOT loaded at runtime. Do not rely on them; edit the `.js` globals.
- Script load order in `index.html` matters: curriculum-data, state, theme, modes,
  checkpoint-registry, checkpoint-core, widgets-unit0..unit12, quiz-data, quiz-engine, router.
- Pre-existing gap (out of scope, unrelated to quizzes): `widgets-unit13.js` and
  `widgets-unit14.js` exist on disk but are NOT referenced in `index.html`.

### 2.2 QUIZ_DATA schema (single source of truth)
```
QUIZ_DATA = {
  micro_practice: { "<video_id>": [ Item x5 ], ... },   // keyed by video_id
  unit_mastery:   { "<exact unit title>": [ Item x30 ], ... } // keyed by unit title
}
Item = {
  id:      string, stable, unique, used as the persistence sub-key
  prompt:  string, may contain KaTeX ($inline$ and $$display$$)
  hint:    string, the general hint revealed by the "Need a hint?" toggle
  answerOptions: [ { text, correct?, rationale } ]   // exactly one correct:true
}
```
- Exactly one option has `correct: true`. Every option (correct and incorrect) carries a
  `rationale`. Correct rationale confirms; incorrect rationale is a first-principles guiding
  question that NEVER names or reveals the correct option (the "Math Confidence Reset").
- `answerOptions` count is 4 in current content (engine tolerates >=2; integrity check
  enforces 4 for mastery, >=3 for micro).

### 2.3 router.js injection points
- Per-video micro practice: inside `buildModuleSection`, within the existing
  `if (unlocked)` block, inside `moduleData.videos.forEach`, appended into each
  `li.video-card` after the watch label. Reads `QUIZ_DATA.micro_practice[video.video_id]`.
  Mounts with `config.id = "micro::" + video.video_id`, host class `video-quiz-host`.
- Unit mastery: in `renderCurriculum`, after `unitSection.appendChild(moduleList)` and
  before `container.appendChild(unitSection)`, so it is the last element of the unit.
  Reads `QUIZ_DATA.unit_mastery[unitData.unit]`. Mounts with
  `config.id = "mastery::" + unitData.unit`, host class `unit-mastery-host`.
- Both guarded by `if (items && items.length)`, so empty/absent keys render nothing.

### 2.4 QuizEngine internal structure (app/js/quiz-engine.js)
- Public: `mount(container, config)` where
  `config = { id, title, intro?, items }`. Renders a launch card only.
- Launch card: title, optional intro, progress line `Progress: X of N complete`, and one
  button labeled `Launch Practice` / `Resume Practice` (partial) / `Review Practice` (all done).
- `openModal(config, live, refresh)`: builds overlay appended to `document.body`, sets
  `body.quiz-modal-open` (scroll lock). Header has title, live `Score: X of N`, and
  `Exit Practice`. Closes via Exit button, Escape key, or backdrop click; close() calls refresh.
- Pacing: opens at first unsolved index (else 0 for review). One question at a time.
- Per question: counter `Question i of N`, prompt, optional `Need a hint?` toggle, options
  as `.quiz-option-row` (button + hidden `.quiz-rationale`). Wrong click -> mark `.incorrect`,
  disable, fill that option's rationale `incorrect` (red), no advance, score unchanged.
  Correct click -> mark `.correct`, lock all, persist, increment score, reveal Next.
- Footer button: `Next Question`, or `Finish` on the last item. Finish -> `renderSummary()`.
- `renderSummary()`: clears body, shows `Practice Complete`, `You answered X of N correctly.`,
  a concluding message, and footer buttons `Retry Quiz` then `Close Practice`.
- `restart()` (Retry Quiz): `ODEState.clearQuizProgress(config.id)`, `live.solved.length = 0`,
  `index = 0`, `updateScore()`, `renderQuestion()`. Restarts in place, modal stays open.
- KaTeX via local `renderMath(el)` calling `renderMathInElement` with delimiters
  `$$`(display) and `$`(inline). Mirrors checkpoint-core.js.
